RAG（檢索增強生成）

ChatGPT 知識停在某個時間點，為什麼還能回答今年的新聞？

LLM 的「知識」是訓練資料的快照——訓練到某個時間點，之後發生的事它都不知道。同樣，公司內部文件、私有資料、最新產品規格，也都不在它的訓練集裡。直接問會得到「我不知道」或更糟，幻覺亂編。

RAG（Retrieval-Augmented Generation）把問題拆兩步：先去外部資料庫「檢索」相關文件，再把這些文件當作 context 塞給 LLM 生成答案。LLM 變成「開卷考」——不靠記憶、看著資料說話。代價是：檢索品質決定答案品質。檢索到爛文件，LLM 也只能基於爛文件回答。

📚
開卷考 vs 閉卷考
閉卷（純 LLM）：腦袋裡記什麼就答什麼，沒記就掰。開卷（RAG）：考前發給你一疊資料，邊翻邊答。資料齊不齊、查得快不快，決定了你考得多好。
✅ 適合
知識頻繁更新、私有/特定領域資料、需要可追溯來源
⚠️ 注意
檢索品質決定一切，垃圾進垃圾出；chunking 與 re-ranking 是 80% 的工程量
❌ 不適合
純創意/推理任務（不需外部知識），或資料量小到能直接放進 prompt

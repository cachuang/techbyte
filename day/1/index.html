JWT vs Session

你知道為什麼現代 API 幾乎不用 Session 了嗎？

當使用者登入，伺服器需要「記住」你是誰。傳統方式是在伺服器儲存一份 Session 紀錄，再給瀏覽器一個 Session ID。但當你有 100 台伺服器時，問題來了——每台都要共享那份紀錄。

JWT 的解法是:把身份資訊直接「編碼」進一個 Token，交給客戶端保管。伺服器只需驗證 Token 的簽章是否合法，不需要查任何資料庫。

🎪
入場手環 vs 接待員記名冊
Session 像活動接待員拿著名冊，每次入場都要查你的名字。JWT 像手環——上面印著你的資訊，任何工作人員掃一下就知道你有沒有資格進去，不需要問總部。
✅ 適合用 JWT 當
微服務、多台伺服器、行動裝置 API
⚠️ 注意
JWT 無法即時撤銷，登出只能靠黑名單或短效期
❌ 不適合
需要即時踢除用戶（如強制下線）的場景

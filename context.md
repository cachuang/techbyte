# techbyte — 開發 Context

## 產品定位

每天 5–10 分鐘，學一個科技概念，做互動驗證，強化記憶。
目標用戶：**已在科技業但想補知識的人**（非新手、非學生）。
內容定調：「你聽過但說不清楚」的概念，有深度、有取捨討論。

---

## 正在驗證的核心假設

> **互動驗證（答題 + 即時解說）真的比純閱讀更幫助記憶**

MVP 不需要多功能，只需要讓這個實驗可以跑起來、收到數據。

---

## MVP 功能範圍

### ✅ 第一版要做

- 每日概念頁面（閱讀）
- 互動答題模組（驗證）
- 答題紀錄（LocalStorage，不需要帳號系統）
- 每日提醒（瀏覽器通知或 Email，後期再做）

### ❌ 第一版不做

- 帳號 / 登入系統
- 社群 / 分享功能
- AI 個人化推薦
- 付費牆
- 知識圖譜視覺化

---

## 使用者流程（Core Loop）

```
READ（閱讀概念）
  → 讀完 60% 以上才能進入答題
QUIZ（互動驗證）
  → 每題選完才能確認，確認後才顯示解說
  → 共 2–3 題，題型輪換
RESULT（結果頁）
  → 顯示分數 + 每題對錯
  → 預告明天概念（強化回訪）
```

## 每個概念的資料結構

```js
{
  day: Number,
  tag: String,           // 分類標籤，如 "API 設計"
  title: String,
  hook: String,          // 開場問句，勾住注意力
  body: String,          // 正文，兩段落，用 \n\n 分隔
  analogy: {
    icon: String,        // emoji
    title: String,
    text: String,
  },
  tradeoffs: [
    { label: String, text: String }  // ✅ 適合 / ⚠️ 注意 / ❌ 不適合
  ],
  questions: [
    {
      id: Number,
      type: String,      // "概念辨識" | "情境判斷" | "錯誤假設"
      question: String,
      options: [
        { id: String, text: String, correct: Boolean }
      ],
      explanation: String,    // 答完後的完整解說
      misconception: String,  // 常見誤解一句話
    }
  ]
}
```

## 答題答案結構（LocalStorage）

```js
answers: [
  {
    qId: Number,
    correct: Boolean,
    unsure: Boolean,   // 使用者選「我不確定答案」時為 true
  }
]
```

## 題型設計（3 種，每個概念輪換）

| 題型 | 說明 |
|------|------|
| 概念辨識 | 測試有沒有記住定義 |
| 情境判斷 | 測試理解使用時機（最重要） |
| 錯誤假設 | 測試能否破除常見誤解 |

---

## 答題選項設計

每題除了 2–3 個正式選項，固定加一個特殊選項：

```
[ ] 我不確定答案
```

- 選這個確認後：顯示正確答案 + 完整解說
- 計分視為答錯（不算分）
- 在結果頁顯示 🤔 圖示
- 未來可用這個數據判斷「哪些概念最多人不確定」

---

## 解說結構（答題後）

```
標頭：
  ✅ 答對了！   （答對）
  ❌ 再想想     （答錯）
  🤔 沒關係，看看解說  （選不確定）

內容：
  explanation  →  為什麼這樣（完整解說）
  misconception → 常見誤解一句話
```

## UI / 設計規格

### 色彩

| 用途 | 色碼 |
|------|------|
| 背景 | `#0a0a0a` |
| 卡片背景 | `#111` |
| 邊框 | `#1e1e1e` / `#2a2a2a` |
| 主文字 | `#f0f0e8` |
| 次要文字 | `#c8c8c0` / `#aaa` |
| 強調色（黃）| `#facc15` |
| 答對（綠）| `#4ade80` |
| 答錯（紅）| `#f87171` |
| 不確定（藍）| `#60a5fa` |

### 字型

- 標題 / 正文：`Georgia`, `Noto Serif TC`, serif
- 標籤 / 按鈕 / badge：`Courier New`, monospace

### 動畫

- 頁面切換：`fadeIn` 0.35s（opacity + translateY）
- 進度點切換題目：`transition: all 0.3s`
- 按鈕 hover：`translateY(-1px)` + yellow glow shadow

## 關鍵互動規則

1. **閱讀進度 > 60% 才能進入答題**（進度條自動計時，防止跳過）
2. **選完選項才能點「確認答案」**（按鈕 opacity 0.3 → 1）
3. **確認後選項鎖定**，顯示對錯顏色 + 解說，才能進下一題
4. **「我不確定答案」是獨立 UI 元素**，視覺上和正式選項區隔（虛線邊框 + 藍色）
